# Guard\<T\>

Conditionally renders children with a **narrowed, non-nullable value**.

Unlike `<Show>`, `<Guard>` narrows the type of `when` so the render-prop
receives `NonNullable<T>` — eliminating null/undefined checks inside children.

## Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `when` | `T \| null \| undefined \| false \| 0 \| ""` | ✅ | Value to check. If truthy, passed to children as `NonNullable<T>`. |
| `children` | `(value: NonNullable<T>) => ReactNode` | ✅ | Render function receiving the narrowed value. |
| `fallback` | `ReactNode` | | Rendered when `when` is falsy. Defaults to `null`. |

## Usage

<Tabs items={['After (flowify)', 'Before (imperative)']}>
<Tabs.Tab>
```tsx
import { Guard } from "flowify";

const user: User | null = useUser();

<Guard when={user} fallback={<Login />}>
  {(u) => <Profile name={u.name} />}
</Guard>
```
</Tabs.Tab>
<Tabs.Tab>
```tsx
const user: User | null = useUser();

{user ? <Profile name={user.name} /> : <Login />}
```
</Tabs.Tab>
</Tabs>

## Why Guard over Show?

<Callout type="info">
`<Show>` only accepts a boolean `when` — it doesn't narrow types.
`<Guard>` is designed for nullable values where you need the narrowed type inside children.
</Callout>

```tsx
// Show: you still need to assert or check inside children
<Show when={user !== null}>
  <Profile name={user!.name} /> {/* ← assertion needed */}
</Show>

// Guard: type is narrowed automatically
<Guard when={user}>
  {(u) => <Profile name={u.name} />} {/* ← u is NonNullable<User> */}
</Guard>
```
