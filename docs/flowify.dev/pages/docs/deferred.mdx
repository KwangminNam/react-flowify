# Deferred\<T\>

Wraps `useDeferredValue()` as a **declarative render-prop** component.

Passes the deferred (lower-priority) version of `value` to children,
allowing expensive re-renders to yield to more urgent updates.

<Callout type="info">
This is a React **Concurrent** feature. Also available as `Concurrent.Deferred`.
</Callout>

## Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `value` | `T` | ✅ | The value to defer. |
| `children` | `(deferredValue: T) => ReactNode` | ✅ | Render function receiving the deferred value. |

## Usage

<Tabs items={['After (flowify)', 'Before (imperative)']}>
<Tabs.Tab>
```tsx
import { Deferred } from "flowify";

const [query, setQuery] = useState("");

<input value={query} onChange={(e) => setQuery(e.target.value)} />

<Deferred value={query}>
  {(deferredQuery) => <SearchResults query={deferredQuery} />}
</Deferred>
```
</Tabs.Tab>
<Tabs.Tab>
```tsx
const [query, setQuery] = useState("");
const deferredQuery = useDeferredValue(query);

<input value={query} onChange={(e) => setQuery(e.target.value)} />
<SearchResults query={deferredQuery} />
```
</Tabs.Tab>
</Tabs>

## When to use

Use `<Deferred>` when a child component is expensive to render and you want
the input to remain responsive. The deferred value lags behind the actual value
during fast updates, preventing jank.
