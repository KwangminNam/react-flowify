# AsyncBoundary

Combines React `<Suspense>` and `<ErrorBoundary>` into a **single declarative boundary**.

Handles both pending (loading) and rejected (error) states, reducing boilerplate
when using async components or `<Use>`.

## Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `children` | `ReactNode` | ✅ | Async content to render. |
| `pendingFallback` | `ReactNode` | | Shown while children are suspended. |
| `rejectedFallback` | `(props: FallbackProps) => ReactNode` | | Shown when an error is thrown. Receives `error` and `resetErrorBoundary`. |

## Usage

<Tabs items={['After (flowify)', 'Before (imperative)']}>
<Tabs.Tab>
```tsx
import { AsyncBoundary, Use } from "flowify";

<AsyncBoundary
  pendingFallback={<Skeleton />}
  rejectedFallback={({ error, resetErrorBoundary }) => (
    <div>
      <p>Error: {error instanceof Error ? error.message : String(error)}</p>
      <button onClick={resetErrorBoundary}>Retry</button>
    </div>
  )}
>
  <Use promise={fetchData()}>
    {(data) => <DataView data={data} />}
  </Use>
</AsyncBoundary>
```
</Tabs.Tab>
<Tabs.Tab>
```tsx
import { Suspense } from "react";
import { ErrorBoundary } from "react-error-boundary";

<ErrorBoundary fallbackRender={({ error, resetErrorBoundary }) => (
  <div>
    <p>Error: {error.message}</p>
    <button onClick={resetErrorBoundary}>Retry</button>
  </div>
)}>
  <Suspense fallback={<Skeleton />}>
    <AsyncContent />
  </Suspense>
</ErrorBoundary>
```
